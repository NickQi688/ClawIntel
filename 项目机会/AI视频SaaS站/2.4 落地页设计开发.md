## 04\. 落地页设计开发

落地页是用户访问网站后看到的第一个页面，也是决定用户是否采取行动的关键页面。一个好的落地页能让转化率提升 3-5 倍以上。

本章将完成落地页的设计与开发工作，包括页面结构、视觉设计、文案撰写和组件判断。同时也会教给大家一些“枯燥”的理论和判断知识。

1\. 为什么落地页决定转化率

1.1 用户来找解决方案，不是看技术

用户访问你的网站，是因为遇到了问题，想找到解决办法。

他们关心的是你的产品能帮他们做什么，而不是你用了什么技术。

很多新手犯的错误是在首屏大谈技术有多先进，算法有多强大，这种做法会让用户立刻关闭页面，因为他们看不懂，也不关心这些内容。

正确的做法是用简单的语言说明产品能帮用户解决什么问题，带来什么好处。

用户只需要知道使用你的产品后，他们的工作会变得更轻松，时间会节省多少，效果会提升多少。

聚焦到我们的视频 SaaS 工具站，视频模型发布时可能会讲到一些技术上的突破和方案，但这些并不是我们的用户会关心的话题，这些我们要能够进行判断。

很多时候，文案比视觉更重要，因为用户更容易识别是否对我有用。

用户打开你的落地页后，会在 5 秒内决定是否继续浏览。如果在这 5 秒内，用户看不懂你的产品是做什么的，或者觉得和自己无关，就会直接关闭页面。

首屏必须在 5 秒内让用户明白三件事：

1️⃣这个产品是做什么的

2️⃣能帮我解决什么问题

3️⃣我为什么应该相信它

所以从这个角度来看，文案的重要程度远大于视觉排版动效这些，前者让他们找到自己和产品的关系进而引发留下来的行为，而后者的目的主要是为了让大家产生信任，不会觉得这是一个虚假钓鱼或粗制滥造的产品。

先给大家看一张落地页模版，了解整体的信息架构，内容翻译自国外设计团队的内容。了解落地页大概构成后，我们会接着讲解一些重要模块并推荐一些对应的组件。

2\. 高转化落地页的 5 个核心板块

首屏一般也叫做 Hero 区域，是落地页最重要的部分，至少包含三个核心元素：大标题、副标题和行动按钮。

大标题必须直接说出产品的核心价值，不要用模糊的词汇，不要玩文字游戏，用最简单的语言告诉用户这个产品能做什么。

比如“10 秒生成 AI 视频”比“智能视频创作平台”更清晰。

这里以小排老师的网站作为案例：

首屏中间核心区域，展示了标题和关键词，聚合信息很容易让用户扫一眼就把所有内容了解（即便没有认真看）。

其中的文案“全球首个无限免费 AI 图像生成器”更是精彩，不仅把产品的特性和噱头表达清楚了，这句话还成为了自媒体们直接“抄答案”的营销话术。

“不要让我思考”的逻辑被套用在了给自己网站做宣传的自媒体身上，不用思考都知道怎么推广这个网站。

首屏区域出现操作组件，往往比一个简单的按钮更容易引导人行动。

按钮的坏处是，用户不知道点击后会发生什么去到哪里，所以视频站、图片站等比较建议是首屏就让用户能操作。

2.2 建立信任感的三种方式

建立信任感要注意不能滥用，否则不仅不会产生信任，还会产生“很假”的感觉。如果新网站没有反馈，则尽量用看起来更加真实且克制的内容和数量，并且尽量在其他模块建立信任。

注意在申请 Creem 支付的时候，一定不要放虚假评价或数据，会被拒绝的。

1️⃣ 使用客户背书，特别是大品牌的客户（用户看到自己领域的老大、竞品都在用你的产品，自己还能落下吗）

2️⃣ 用数据说话：注意不要是死数据，是动态的会变化的数据更能增加可信度。

比如 evolink 右上角的API状态和成功率，他是变化的数值，给用户感觉更加真实：

3️⃣ 真实用户评价：现在这个组件有点被滥用了，很多数据都不真实，用户的可信度也下降了，所以在这个前提下，更要注重用户和内容的真实性。且有效的评价内容 不要用过于速度快的滚动效果，因为那根本不是给人看的。

比如现在留下这种 tweet 卡片，可以点击过去看原文，这种就很建立真实和信任。

2.3 展示核心功能的正确方式

只展示 3-8 个最核心的功能，每个功能说明能为用户带来什么好处。不要列举所有功能，用户会困惑。每个功能配一个演示动图或截图，文字说明不超过两句话。核心功能放前面，次要功能放后面或单独页面。

2.4 定价方案的设计技巧

定价页面要清晰明了，让用户一眼看懂不同方案的区别。最常见的方式是三栏对比，基础版、专业版、企业版并排展示。

提供免费试用、退款保证或按月付费，降低用户心理门槛。

提供企业定制选项，一般是为了证明产品能力或有高净值企业合作来辅助证明可信度，而不是一定期望用户购买的选项。

几个注意点：

1️⃣ 提供年付折扣优惠

2️⃣ 提供最推荐选项（不能是最便宜或最贵的那个）

我个人是建议优先展示月付，大折扣优惠吸引年付，也同时提供新人一次性积分以便顺利完成转化。真实用户其实并不喜欢订阅，因为还没开始用你的产品就要每个月扣费，每个人都会犹豫的，但一次性付费就轻松了，等用户真觉得好用会再来订阅的。

FAQ 部分选择 5-8 个用户最关心的问题，用简短文字给出答案。典型问题：需要信用卡吗？可以随时取消吗？数据安全吗？

大多数 FAQ 都是放在页面最底部，页面最后再次放置行动召唤按钮，让用户看完整个页面后能立刻找到行动入口。

转化率低，先检查板块顺序是否正确，首屏是否有 CTA 按钮。不要把所有信息都塞进落地页，保持简洁。

还有就是需要注意移动端的适配，现在很多用户都通过移动端搜索和使用产品，我之前分析很多产品的流量构成，也有很多移动端占比很高，甚至有些网站因为没有注重移动端体验优化，导致了大量用户流式的情况也很多。

现在的很多组件都已经做好了移动端的适配，特别需要注意的是我们用 AI 编程自定义实现的那些组件，要特别测试一下移动端效果最佳。

如何检查移动端适配，可在浏览器中查看：

鼠标右键菜单选择“检查”或“Inspect”，或在顶部菜单 View 里选择 Developer Tools

此时右侧弹出一个面板，不用管其他，点击面板左上角的按钮即可查看移动端效果，回到桌面端只需再次点击即可。

3\. 围绕关键词构建内容

我们前面有提到做视频站，可以先从一个视频模型词出发，也可以多很多模型聚合，甚至找一些视频方向垂直需求做长尾关键词。

如果是要做长尾的视频需求，关于如何找需求如何找词，可以参考热词快站航海的手册：《 热词挖掘方法与实战任务 》里面有相关的方法和工具介绍。

这门这里选定了关键词后，整个首页都要围绕关键词为核心展开，这方面是为了能让 Google 等搜索引擎了解我们是做什么的，进而推荐给有使用需求的用户。

3.2 在内容中自然融入关键词

核心关键词出现在页面标题、H1 标题、首段文字、至少一个 H2 标题，不要刻意堆砌，保持自然，用户体验与 SEO 都重要。

一般来说，可以把首页核心关键词的密度控制在总词汇的 2%~5% 左右。可以利用 AITDK 等工具查看，也可以让 AI 帮你控制。

Tips：有时候不好加更多词了，可以增加到 FAQ 的问题回答中，这样既不会降低用户体验（全都是关键词），也不会对搜索引擎不友好。

落地页开发不需要从零开始写代码，使用现成的 UI 组件库可以大大提高效率。

而且现成的组件库都考虑了移动端布局和各种复杂的边界场景。

下面介绍三个我自己最常用的组件库：shadcn/ui、Magic UI 和 Animate UI。这些组件库提供了丰富的预制组件，可以直接用来实现前面讲到的各个板块。

4.1 shadcn/ui - 简洁高效的基础组件库

shadcn/ui 是一个基于 React 的组件库，提供了大量简洁、现代的 UI 组件。它的特点是组件代码直接复制到项目中，你拥有完全的控制权，可以自由修改样式和功能。

这部分组件库已经作为基础组件落实在我们的项目模版中，正常页面的布局创建都会使用这些基础组件。大部分复杂业务逻辑的实现，也往往是这些基础组件的拼装组合。

日常使用中，我们不必再查询相关组件方案，但是作为学习认识不同的基础组件，还是大有帮助。

网址： https://ui.shadcn.com/

4.2 Magic UI - 炫酷的动效组件库

Magic UI 专注于提供带有精美动效的组件。如果你希望落地页更有视觉冲击力，Magic UI 是很好的选择。它包含了许多酷炫的动画效果组件，比如渐入动画、悬停效果、滚动触发动画等。

Magic UI 适合用在首屏区域制造视觉吸引力，比如带有动态背景的 Hero Section。功能展示板块可以用它的动画卡片，让每个功能在用户滚动到时候有渐入效果。客户评价区域可以用它的自动滚动轮播组件。

网址： https://magicui.design/

他们的付费产品包含了非常多的精致模板和组件： https://pro.magicui.design/ 是一个很好的模仿学习方案。

4.3 Animate UI - 专业的动画组件库

Animate UI 提供了更丰富的动画预设和更精细的动画控制。它不仅有现成的动画组件，还提供了动画工具函数，让你可以给任何元素添加动画效果。

Animate UI 适合需要更复杂交互效果的场景。比如首屏的文字逐字出现效果，功能展示的图片跟随鼠标移动，定价方案的对比高亮动画。它还提供了滚动视差效果，让落地页更有层次感。

网址： https://animate-ui.com/

同样的，他也有付费方案： https://pro.aceternity.com/

这里面的模版非常丰富，Pro 版本的组件也更加精致，教练之前还团购过此产品，推荐购买使用。

其他推荐的一些组件库：

提供更多特定场景细节交互的组件（如 Color Picker、Kanban、Tour等）： https://www.diceui.com/

专注在动效动画类组件，如果需要可以使用： https://reactbits.dev/

4.4 如何选择合适的组件

选择组件要结合内容需求和品牌风格：

不要在一个页面上混用太多不同风格的组件，保持视觉统一。

如果产品定位是专业严肃的，以 shadcn/ui、DiceUI 为主即可，少用花哨动效。

如果产品定位是创新炫酷的，可以多用 Magic UI 和 Animate UI，甚至加上 Reactbits 都可以。

不知道选哪个组件库，建议先从你喜欢的竞品模仿开始，也可以模仿学习 Animate UI Pro 和 Magic UI Pro 上的页面模版。

这里教练建议，如果想要做出更好的作品，一定要多看优秀作品，培养自己对产品的“审美”，注意不仅是交互视觉层面，还要能识别和判断好产品，推荐去 Mobbin 上看，这里有很多优秀产品的实际页面截图和流程截图，内容都是上线过经过市场验证的，而不是一些设计师社区的飞机稿。

网址： https://mobbin.com/

如果你的文案是经过精心打磨过的，你觉得视觉效果只要达到及格就能使用，也可以使用偷懒的方法，让 AI 帮你根据文案内容设计适合的组件，此时考察的就是 AI 编程模型的审美和前端能力了。

当前时间点下，前端能力表现最好的模型是 Gemini 3 （pro 跟 flash 都可以），所以如果你会安装使用 Antigravity 的话，建议用户使用这个模型去开发组件，效果会更好。

另一种方式，你也可以让 AI 基于全部文案先生成一个落地页，再慢慢调整各部分的组件，我测试了一下 GLM 4.7 的能力，虽然让他遵循内置组件进行开发，但还是有很多自作主张的自定义组件。

落地页是需要一定功夫打磨和优化的，一旦打磨好了，就要尽量少修改，避免 Google 对我们原本已经爬取的关键词进行调整，那样会造成流量的损失。
