## 01\. 支付接入与积分系统

💡

本章将指导你完成支付系统的完整配置流程。你会学到如何制定合理的定价策略，在 Creem 平台创建商品，配置支付接口，以及测试整个支付流程。完成本章后，你的网站将具备完整的收费能力，用户可以购买积分包或订阅套餐来使用视频生成服务。

1.1 定价前的三个核心问题

在开始设置价格之前，你需要先回答三个基本问题。第一个问题是你的成本是多少，也就是每次调用 AI 模型生成视频需要支付多少 API 费用。第二个问题是你想要多少利润，一般建议目标毛利率设定在 50% 到 70% 之间，同时要预留未来促销活动的折扣空间，比如黑色星期五或圣诞节期间的优惠。第三个问题是用户愿意付多少钱，这需要你参考市场上同类产品的价格和用户对价值的感知。

确定价格有两种常用方法。第一种是参考同行定价，这是最简单的方式，你可以找几个类似的视频生成网站，按照他们的价格来设定，甚至后续的促销活动也可以模仿学习。第二种是基于成本价的利润率定价，先计算出你的成本，再根据目标利润率来倒推售价。

本教程采用第二种方法来演示，整体定价在视频站中是偏低的，主要是为了教学演示。你需要根据自己的实际情况来设定最终价格。

本模板使用 Veo 3.1 Fast Lite 作为基准模型。根据 Evolink 平台的价格，1 积分约等于 0.1 元人民币，也就是约 0.014 美元。如果生成一个视频消耗 10 积分，那么成本就是 1 元人民币，约 0.14 美元。如果你的售价是成本的 2.5 倍，也就是 0.35 美元，那么毛利率约为 60%。

这里需要强调的是，你必须根据自己选择的 AI 模型和 API 供应商来重新计算成本。不同模型的价格差异很大，直接套用示例数据可能导致亏损或定价过高。

模板已经为你准备了一套完整的定价方案，包含 9 个商品，分为三类。第一类是一次性积分包，共 3 个，分别是 Starter 包 14.90 美元、Standard 包 39.90 美元、Pro 包 99.90 美元，有效期都是 1 年。第二类是月付订阅，共 3 个，分别是 Basic 月付 9.90 美元、Pro 月付 29.90 美元、Ultimate 月付 79.90 美元。第三类是年付订阅，共 3 个，分别是 Basic 年付 99 美元、Pro 年付 299 美元、Ultimate 年付 799 美元，年付相当于月付乘以 10，等于买 10 个月送 2 个月，统一享受 17% 的折扣。

3.2 填写商品名称和描述

名称和描述是用户在购买时会看到的核心信息。商品名称需要填写与网站中显示的名称一致，比如 Basic Monthly 或 Starter Pack。描述部分可以写一些激励用户付款的内容，比如说明可以随时申请退款，或者强调这个套餐最适合什么样的用户。

价格类型分为两种。Single Payment 是一次性商品，积分包就选择这个类型。Subscription 是订阅商品，按月或按年订阅的商品选择这个类型。

关于税费的处理，一般建议选择含税价。因为不同国家的税收政策不一样，如果选择不含税，用户点击进来后发现含税价格更高，可能就不愿意付款了。这些税费成本一般在你的利润范围内可以覆盖，所以建议选择含税价来提高付款转化率。

Creem 现在新增了一个非常实用的功能叫 Abandoned Cart Recovery，也就是弃单挽回。勾选这个选项后，如果用户放弃付款，过一段时间 Creem 会自动发邮件提醒用户完成付款。这个功能可以有效提升付款转化率，如果你没有配置其他自动化营销工具，建议勾选上。

这是用户收到的催促付款邮件样式，看起来很专业。

基本上填写完名称、描述、价格和类型这些信息就足够了。商品图片是可选项，影响不大。确认所有信息都没有问题，特别要仔细检查价格是否正确，然后点击 Create Product 按钮。

这里要特别注意订阅商品的计费周期。创建月付订阅时要选择 Monthly，创建年付订阅时要选择 Yearly，千万不要选错。

按照同样的方法，把定价方案文档中要求的所有商品都创建完成。创建完成后，在商品列表页面仔细检查一遍，确认每个商品的名称、价格和订阅周期都准确无误。

4.1 理解配置文件的变化

完成商品创建后，你就拿到了所有商品的 Product ID。接下来需要将这些 ID 配置到代码中。

旧版模板是在.env.local 文件中配置商品 ID，但最新版本的模板已经改变了配置方式。现在所有价格和积分的配置都在 src/config/pricing-user.ts 文件中。

如果你使用的是旧版模板，需要先清理一下。打开.env.local 文件，如果看到下图中这些以 CREEM\_PRODUCT 开头的变量，需要全部删除，因为这些变量已经弃用。

打开 src/config/pricing-user.ts 文件，这个文件中已经包含了详细的使用说明。你也可以访问官方文档 https://docs.videofly.app/docs/config/price 查看更详细的配置教程。

回到 Creem 的商品列表页面，每个商品后面都有一个复制按钮，点击可以复制该商品的 Product ID。

复制 Product ID 后，打开 src/config/pricing-user.ts 文件，找到对应的商品配置项，将 ID 粘贴进去。

这里需要注意，文件中的其他信息，比如价格、积分数量、过期时间等，在前面步骤中应该已经让 AI 帮你修改过了。如果发现这些信息与你的定价方案有差异，建议让 AI 再检查一遍代码，确保所有配置都正确。

4.5 配置积分包购买权限

这里有一个特别的配置需要说明。一般来说，用户都不太喜欢订阅，特别是对第一次访问的网站，在还没有建立任何信任之前就要求每个月固定扣费，转化率会比较低。相对来说，一次性积分包更容易促成首次购买。

因此模板代码支持设置积分包的购买权限。你可以设置最便宜的积分包允许免费用户购买，而其他积分包仅限付费用户购买。这样可以降低新用户的购买门槛，同时也能引导老用户购买更大的积分包。

5.1 关于测试环境的说明

按照标准流程，我们应该先在测试模式下进行本地开发测试。这个过程需要安装 ngrok 工具来建立本地 webhook 服务，然后使用测试信用卡进行支付测试。但这个过程比较复杂，为了简化操作，我们将直接进行线上环境的配置。

如果你对测试环境开发感兴趣，可以访问 Creem 官方文档 https://docs.creem.io/getting-started/test-mode 学习相关内容。

进入 Creem 后台的 API & Webhook 页面，直接复制显示的 API Key。

打开本地项目的.env.local 文件，找到 CREEM\_API\_KEY 这一行，将刚才复制的 API Key 粘贴进去。

在配置 Webhook 之前，先简单解释一下它的作用。Webhook 就像是支付平台给你网站发的自动通知短信。当用户完成付款后，Creem 会立即向你的网站发送一条消息，告诉你某某用户刚刚支付了多少钱买了什么套餐。你的网站收到这个通知后，就会自动给用户充值积分，整个过程只需要几秒钟。

配置 Webhook 就是告诉 Creem 你网站的通知地址，就像告诉快递员你的收货地址一样，这样 Creem 才知道把付款通知发到哪里。

在 API & Webhook 页面，点击创建 Webhook 按钮。

首先输入一个名字，可以随便起，比如 Production Webhook 或者 Main Webhook。

然后需要输入 Webhook URL。因为我们使用了 Creem 和 Better Auth 的插件方式集成，Webhook URL 的格式是固定的，按照下面的格式填写。

```
https://你的域名.xxx/api/auth/creem/webhook
```

把其中的“ 你的域名.xxx “替换成你的实际域名，比如 example.com ，完整的 URL 就是 https://example.com/api/auth/creem/webhook。

5.5 使用测试域名的技巧

这里有一个不用正式域名就能测试的方法。Vercel 部署时会自动分配一个临时域名，我们可以把它当做测试域名使用。

你可以创建两个 Webhook，一个用正式域名，一个用 Vercel 分配的临时域名。这样就可以在测试域名下进行支付测试，避免在正式域名上测试时被用户看到可能存在的问题。

需要注意的是，如果使用 Vercel 临时域名测试，还需要在 Google 登录配置中添加这个域名，否则用户无法通过 Google 账户登录。

比如下图中，我就是用测试域名进行支付测试的，这样可以防止用户在正式网站上不小心遇到还未修复的问题。

保存 Webhook 后，你会看到它已经激活。

点击进入 Webhook 详情页面。

找到 Webhook Secret 这一栏，先点击显示按钮，然后点击复制按钮。

打开本地项目的.env.local 文件，找到 CREEM\_WEBHOOK\_SECRET 这一行，将刚才复制的 Webhook Secret 粘贴进去。

这一步非常重要。你刚才修改的是本地的.env.local 文件，但线上网站使用的是 Vercel 后台配置的环境变量。所以你必须将这两个新配置的变量同步到 Vercel，并重新部署，线上网站才能正常工作。

登录 Vercel 后台，进入你的项目，点击 Settings 选项卡，然后在左侧菜单找到 Environment Variables。

找到 CREEM\_API\_KEY 和 CREEM\_WEBHOOK\_SECRET 这两个变量，点击 Edit 按钮，将新的值粘贴进去，然后保存。保存后 Vercel 会提示你需要重新部署才能生效，点击重新部署按钮即可（忘记截图 右下角会有弹窗提示）。

前面我们制定了一份完整的价格方案文档，现在需要让前端页面的价格展示与这份文档保持一致。你可以用一句话让 AI 帮你完成这个工作，比如“基于 docs/spec/PRICING\_REFERENCE.md 文档，修改价格组件进行适配”。

在 AI 完成基础修改后，你可以继续优化价格页面的展示效果。这里最重要的是文案的清晰度和吸引力。

一般来说，默认会显示月付订阅页面，因为月付的价格看起来更低，用户更容易接受。年付页面的标签上会显示折扣信息，比如“省 17%”，用来吸引用户点击查看。

如果用户不愿意订阅，可以切换到 One-time 标签页查看一次性积分包。

正常情况下，支付系统有很多正常流程和异常流程需要测试，你可以让 AI 帮你写一份完整的测试用例。但对于新手来说，我们只需要测试一次积分包购买和一次订阅购买就基本足够了。只要有一个商品能够成功走通整个流程，其他商品通过 AI 检查代码逻辑基本就能确认没有问题。

因为我们是在线上环境测试，需要使用真实的银行卡或信用卡进行支付。为了避免资金损失，有两个可选方案。

第一个方案是测试后在 Creem 后台进行退款。进入订单详情页面，点击 Refund 按钮即可退款。

第二个方案是创建折扣码进行支付。在 Creem 后台创建一个 100% 折扣的优惠码，这样测试时就不需要实际支付了。

在网站上选择一个商品，点击购买按钮，会跳转到 Creem 支付页面。如果你创建了折扣码，可以在支付页面输入折扣码。

填写支付信息时，地址选择 China 不需要输入具体地址。完成支付后，系统会自动跳转回你的网站。

同时 你的邮箱也会收到邮件。

支付成功后，进入网站的积分页面，应该能看到积分已经充值到账，并且能在积分历史记录中看到这笔充值记录。如果积分正常到账，说明整个支付流程配置成功。

注意，如果你用自己银行卡测试过订阅，一定记得去后台给自己取消订阅，否则下个月会自动扣款。

至此，支付系统的完整配置就全部完成了。你的网站现在已经具备了完整的收费能力，用户可以购买积分包或订阅套餐来使用视频生成服务。
