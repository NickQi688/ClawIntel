## 05\. 代码上传与部署

本章将指导你完成代码上传和网站部署的全部流程。你会学到如何将本地开发好的代码推送到 GitHub 仓库，然后通过 Vercel 平台部署上线，最后绑定你自己的域名。完成这一章后，你的网站就能正式对外提供服务了。

本地代码调试完成后，需要上传到 GitHub 仓库。这一步是为了让 Vercel 能够读取你的代码并进行部署。我们提供三种上传方式，你可以选择最适合自己的方法。

2.1 使用 GitHub Desktop 客户端上传

这是最简单的方式。因为我们之前就是用 GitHub Desktop 克隆的代码，现在上传时不需要处理任何权限问题。

打开 GitHub Desktop，软件会自动检测到你修改过的文件。在左下角的输入框中填写本次修改的说明，比如“完成首页视频组件配置”，然后点击右上角的 Push origin 按钮即可完成上传。

如果你习惯在 Cursor 中工作，可以直接在编辑器内完成上传。点击左侧边栏的源代码管理图标，就能看到所有修改过的文件。填写提交说明后点击提交按钮，代码就会上传到 GitHub。

这里不再详细讲解 Git 命令行的使用方式。如果你对命令行操作感兴趣，可以查看以往的航海手册或向 AI 询问相关指令和注意事项。

这是最省心的方式。你可以直接在 Cursor 中对 AI 说“帮我上传代码到 GitHub”，AI 会自动完成整个上传流程，还会帮你写好提交说明。如果遇到账号权限问题，AI 也能引导你解决。

AI 上传的另一个好处是提交记录会更规范。当你需要回退到某个历史版本时，清晰的提交记录能帮你快速找到对应的代码状态。

代码上传到 GitHub 后，就可以开始部署了。Vercel 是一个专门为前端项目提供部署服务的平台，它会自动构建你的代码并生成可访问的网站。

登录 Vercel 后台，点击右上角的 Add New 按钮，选择 Project 选项。

在导入项目页面，选择从 GitHub 导入。

找到你刚才上传代码的仓库，点击 Import 按钮。

这一步需要将本地的环境变量复制到 Vercel。打开本地项目中的.env.local 文件，全选复制所有内容。然后在 Vercel 的环境变量输入框中粘贴，系统会自动识别并批量填入所有变量。

这里有一个重要提醒。本地环境变量中的一些配置需要改成线上版本。比如 localhost:3000 要改成你的实际域名，http:// 要改成 https://。这是因为线上环境和本地开发环境的访问方式不同。

点击 Deploy 按钮后，Vercel 会开始构建你的项目。这个过程通常需要几分钟时间。构建成功后，Vercel 会为你生成一个临时域名，你可以通过这个域名访问网站。

部署失败很正常，这时候不用慌，我们可以点击一键复制所有错误代码，然后发给 AI，让 Claude code 帮我们进行修改。

临时域名虽然能用，但不够专业。接下来我们要绑定自己的域名，让网站看起来更正规。

进入项目设置页面，点击 Domains 选项卡。

点击 Add 按钮准备添加域名。

在输入框中填入你的域名。这里建议使用不带 www 的纯域名，比如 example.com 而不是 www.example.com 。

关于 www 的处理建议。如果你希望所有访问都统一显示为 example.com ，可以在 Cloudflare 中设置 CNAME 重定向。具体做法是添加一条 CNAME 记录，将 www 指向 @，这样用户访问 www.example.com 时会自动跳转到 example.com 。

Vercel 提供了一键配置功能。如果你的域名托管在 Cloudflare，点击 Add 按钮后会出现授权提示。授权后 Vercel 会自动在 Cloudflare 中添加所需的 DNS 记录，整个过程不到一分钟。

配置完成后，等待几分钟让 DNS 解析生效。你可以通过访问自己的域名来确认网站是否正常运行。

到这里，你的网站就正式上线了。接下来可以开始考虑如何获取流量，提升转化率，让更多用户使用你的产品。
