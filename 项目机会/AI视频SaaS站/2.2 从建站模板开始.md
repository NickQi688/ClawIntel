8.1 为什么需要域名邮箱

域名邮箱就是使用你自己域名作为后缀的邮箱，比如你的域名是 myaisite.com，那么域名邮箱就是 support@myaisite.com 这样的格式。

使用域名邮箱有两个好处：

第一是显得更专业，当用户看到邮件来自你的域名邮箱而不是普通的 Gmail 或 QQ 邮箱时，会更信任你的网站。

第二是方便管理，你可以创建不同用途的邮箱，比如 support@myaisite.com 用于客服，billing@myaisite.com 用于财务，让工作分工更清晰。

第三个就是我自己经常用的，很多当前网站的服务，可以直接用域名邮箱注册，避免用自己的主账号浪费免费的额度。（这一点后面会讲到怎么用）

Cloudflare 提供免费的邮件转发服务，可以把发送到域名邮箱的邮件自动转发到你的个人邮箱，这样你不需要额外登录域名邮箱查看邮件，所有邮件都会转发到你常用的邮箱中。

在 Cloudflare 控制面板找到你的域名，点击域名进入管理页面。

然后在左侧菜单中找到电子邮件或 Email Routing 选项，点击进入电子邮件路由页面。

第一步需要配置DNS，因为域名解析到 Cloudflare 上，这一步可以一键完成，点击“添加记录并启用”即可。

目标地址下面，是你最终接受邮件的真实地址，一般是你的登录账号，你也可以添加其他的邮箱。

如果没有目标地址

则点击“添加目标地址”，在自定义地址栏中输入你想要的域名邮箱前缀，比如输入 contact，系统会自动补全成 contact @你的域名.com。

在目标地址栏中输入你的个人邮箱，这个邮箱是用来接收转发邮件的，建议使用你最常查看的邮箱，比如 Gmail 或企业邮箱。

点击保存后，Cloudflare 会向你的目标邮箱发送一封验证邮件。

打开邮件，点击验证链接完成邮箱验证，验证完成后，转发规则就生效了。

来到“路由规则”页面，在“自定义地址”中点击创建地址，然后填入相关的信息保存。

可以看到此时自定义域名邮箱已经设置成功，确保状态为开启即可。

配置完成后，建议立即测试域名邮箱是否正常工作。用你的另一个邮箱或者朋友的邮箱，发送一封测试邮件到刚才配置的域名邮箱，比如 support@你的域名.com。

等待几分钟后，检查你的目标邮箱是否收到转发的邮件，如果收到了，说明域名邮箱配置成功。如果没有收到，检查转发规则是否已经验证，同时查看目标邮箱的垃圾邮件文件夹，有些邮件服务商可能会把转发邮件标记为垃圾邮件。

如果收不到验证邮件，首先检查你输入的目标邮箱地址是否正确。然后查看垃圾邮件文件夹，验证邮件可能被误判为垃圾邮件。如果还是找不到，可以在 Cloudflare 页面点击重新发送验证邮件。

域名邮箱只能接收邮件，不能直接发送邮件，如果你需要用域名邮箱发送邮件，需要在你的个人邮箱中设置发件人别名，这个功能大部分邮箱服务商都支持，具体设置方法可以搜索你使用的邮箱服务商的帮助文档。

到这里，域名的注册和基础配置就全部完成了。

现在你拥有了一个可以正常解析的域名，并且配置好了域名邮箱。

下一章我们会使用这个域名来部署网站。

Resend 是我们的邮件服务，用户注册激活、找回密码邮件还有营销邮件，都可以用它实现。不过它的免费版只支持一个域名绑定，所以如果你要做多个网站，这里强烈建议使用域名邮箱注册，可详见域名注册与解析部分。

访问 https://resend.com 按照下面的操作进行

复制API Key，添加到.env.local，并填写默认邮箱

```
RESEND_API_KEY="your-resend-api-key"
EMAIL_FROM="support@yourdomain.com"
```

还需要在 Resend 中绑定域名（如果此时还没有域名，可以注册了再回来绑定）：

启动开发服务器：

```
pnpm dev
```

看到ready started server on 0.0.0.0:3000说明启动成功。访问 http://localhost:3000 查看网站。

测试功能：检查首页显示、Google登录、数据库连接（登录后刷新页面看用户状态是否保持）、浏览器Console是否有错误。

设置管理员账号，在.env.local 中设置邮箱，如有多个邮箱用英文逗号分隔：

```
# admin account
ADMIN_EMAIL="aaaaaa@gmail.com, bbbbb@gmail.com"
```

后台地址路径： https://你的域名.xxx/admin 加上 /admin 即可。

你可以让 AI 给你增加更多的管理员数据面板和功能，方便你了解自己的网站状况。
