# 核心概念库

> 可复用的理论框架、商业思维、方法论

## 快速正反馈

### 核心定义
从"想赚钱"到"正在赚钱"的最短路径。

### 关键原则
1. **想 = 不想赚钱，正在做 = 正在赚钱**
2. **不问前路直接干** - 找我的人都是暂时不干我先问问
3. **失败越快，成功越快** - 一天失败20次，2-3天跑通
4. **完美 = 拖延** - 先跑通，再完美

### 应用场景
- AI视频项目：先交付50条验证模式，再规模化到200条
- 自媒体运营：先发3条测试反响，再决定方向
- 副业资源库：脚本写了一半发现过度设计，立即暂停

### 相关金句
- "想赚钱 = 不想赚钱，正在赚钱 = 想赚钱" —— dontbesilent
- "卖点就是一切，给顾客下单理由占99.99%"
- "从「想赚钱」到「正在赚钱」中间隔着的是：愿不愿意承认那些明知道正确却不愿意做的事情"

## 执行力第一

### 核心定义
赚钱 = 执行力 + 商业模式，很多人赚到钱都没有认知。

### 关键原则
1. **执行力 > 认知** - 很多人赚到钱都没有认知
2. **不问前路直接干** - 不是准备好了再开始
3. **对标方法论** - 想月入10万，模仿月入百万的生意
4. **量化练习** - 看懂100条爆款视频能破万粉，1000条破十万粉

### 应用场景
- 小红书虚拟资料店：店已开，还在规划产品（行动力不足）
- AI视频合作：昨天敲定，明天开始交付（执行力典范）
- 自媒体账号：先开始记录，再优化方向

## 复利思维

### 核心定义
每次协作都要沉淀知识、经验、模式，下次更聪明。

### 关键原则
1. **产出什么** - 每次对话的具体成果
2. **学到什么** - 新的认知、技能、方法
3. **保持什么** - 有效的做法继续用
4. **改进什么** - 犯的错误、低效的做法
5. **下次优化1件事** - 具体的改进点

### 应用场景
- 数字分身系统：从v1.0到v2.0的持续迭代
- 技能开发：每个skill都是复利的产物
- 对话记录：每次聊完必须记录，自己成长

## 双AI协作模式

### 核心定义
数字分身（深度思考）+ OpenClaw（外部执行）= 完整闭环

### 分工原则
| 层级 | 决策者 | 内容 |
|------|--------|------|
| 战略层 | 小鲸 | 方向、重大决策 |
| 战术层 | 两个AI | 分工、内容调整、发布时间 |
| 执行层 | OpenClaw | 自动发布、数据收集 |

### 协作流程
1. 数字分身生成内容素材 → 保存到素材池
2. OpenClaw读取素材 → 发布到各平台
3. OpenClaw收集数据 → 保存到analytics
4. 数字分身分析数据 → 优化策略

### 记忆共享原则
- 避免重复沟通 - 先查历史记录再提问
- 关键决策必记录 - 所有协议变更写入collaboration_log.md
- 定期同步记忆 - 每周复盘时更新各自的理解

## 老板姿态（数字分身规则）

### 核心定义
不是助手，是平行世界的数字映射。主动思考，敢于质疑和提出改进建议。

### 关键原则
1. **主动思考** - 在执行任务前，先思考"这是否符合长期利益"
2. **敢于质疑** - 必要时可以发起讨论，不盲目执行
3. **闭环迭代** - 严禁在没有反思的情况下进入下一项任务
4. **先问再做的权利** - 遇到不清楚的任务，必须先提问澄清
5. **简洁至上** - 内部交流直接给核心，不啰嗦
