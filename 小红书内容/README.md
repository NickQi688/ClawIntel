# 小红书内容管理中心

> 📱 专门管理小红书运营内容的文件夹  
> 🔄 通过 Obsidian Git 同步到所有设备

---

## 📁 目录结构

```
小红书内容/
├── 01-已发布/          # 已发布的笔记存档
├── 02-待发布/          # 准备发布的笔记内容
├── 03-封面素材/        # 封面图片、设计稿
├── 04-文案模板/        # 文案模板、策略文档
└── 05-数据追踪/        # 发布数据、分析报告
```

---

## 📂 各目录说明

### 01-已发布
存放已发布到小红书的笔记内容
- 发布日期命名：`2026-02-05_第1篇_AI做封面.md`
- 包含：文案、配图、标签、发布链接
- 方便回顾和复盘

### 02-待发布
存放准备发布的笔记
- 即将发布的笔记草稿
- 排版好的完整内容
- 等待合适时机发布

### 03-封面素材
所有封面图片和设计素材
- 原始图片（.jpg, .png）
- 设计源文件（.psd, .sketch）
- AI生成的prompts和中间版本

### 04-文案模板
文案策略和模板文档
- 爆款标题公式
- 正文结构模板
- 互动话术库
- 运营策略文档

### 05-数据追踪
发布后的数据记录和分析
- 每篇笔记的数据截图
- 数据分析表格
- 爆款规律总结

---

## 🚀 快速使用指南

### 发布新笔记流程

```
1. 在 04-文案模板/ 找策略或创建新内容
2. 在 03-封面素材/ 准备封面图片
3. 在 02-待发布/ 完成排版和文案
4. 发布到小红书
5. 移动到 01-已发布/ 并记录数据
6. 在 05-数据追踪/ 记录表现
```

### 文件命名规范

**已发布笔记：**
```
日期_序号_主题_数据.md
例如：2026-02-05_01_AI做封面_阅读500点赞20.md
```

**封面图片：**
```
封面_日期_序号_主题.jpg
例如：封面_2026-02-05_01_AI做封面.jpg
```

**数据记录：**
```
数据_日期_第X周.md
例如：数据_2026-02-05_第1周.md
```

## 💡 使用技巧

1. **同步检查**：每次发布后在另一台电脑检查是否同步成功
2. **定期整理**：每周整理一次，把待发布的移到已发布
3. **备份重要数据**：数据追踪文件记得备份到云端
4. **复用素材**：好的封面设计可以改改颜色复用
