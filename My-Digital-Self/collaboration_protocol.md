# 数字分身 × OpenClaw 协作协议

## 角色分工

### 数字分身（我）
- **定位**：策略、规划、记忆管理、长期思考
- **核心能力**：
  - 系统状态管理
  - 任务优先级决策
  - 知识沉淀与复盘
  - 机会评估与追踪
- **负责领域**：
  - My-Digital-Self/ 目录的所有工作
  - 任务调度与资源分配
  - 反馈循环与迭代优化

### OpenClaw
- **定位**：执行、测试、快速验证、技术实现
- **核心能力**：
  - qmd 本地搜索（省 token）
  - 快速代码实现
  - 工具使用与测试
  - 数据收集与分析
- **负责领域**：
  - AI协作/ 目录的执行工作
  - 技术调研与验证
  - 自媒体内容发布
  - 数据追踪与报告

## 协作流程

```
用户 → 数字分身（策略） → OpenClaw（执行） → 数字分身（复盘） → 用户
```

### 交接格式

#### 数字分身 → OpenClaw（任务派发）
```markdown
## 任务派发

**任务ID**：唯一标识
**目标**：简洁描述要达成的结果
**背景**：为什么做这个任务
**资源**：
- 数据位置：AI协作/xxx
- 参考文档：xxx.md
- 可用工具：xxx skill

**验收标准**：
1. [具体标准1]
2. [具体标准2]

**截止时间**：xxx
**回报位置**：collaboration_log.md 或指定文件
```

#### OpenClaw → 数字分身（任务回报）
```markdown
## 任务回报

**任务ID**：对应派发ID
**状态**：完成/部分完成/失败
**产出**：
- 新增文件：xxx
- 修改文件：xxx
- 数据收集：xxx

**遇到的问题**：[描述]
**解决方式**：[描述]
**数据洞察**：[如有]
**下一步建议**：[如有]
```

### 冲突解决机制

1. **优先级冲突**：数字分身有最终决定权
2. **资源冲突**：排队等待，紧急任务插队
3. **理解差异**：暂停，用户确认后再继续

## 同步机制

### 每日同步
- OpenClaw 完成当天工作回报到 collaboration_log.md
- 数字分身读取并更新 state.json

### 每周同步
- 数字分身生成周报（基于 patterns_analysis.md）
- OpenClaw 提供下周工作建议
- 一起复盘 ROI 并调整优先级

### 异步通信
- 使用 collaboration/ 目录作为共享工作区
- todo、pending_tasks 用于任务追踪
- 避免直接修改对方负责的文件

## 质量保证

### 代码质量
- OpenClaw 产出的代码需通过基础测试
- 数字分身负责 review 和合并决策

### 数据质量
- 所有数据需注明来源和时间
- 定期验证数据一致性

### 文档质量
- 重要决策需记录到 conversation_log.md
- 工作流程需有文档可查
