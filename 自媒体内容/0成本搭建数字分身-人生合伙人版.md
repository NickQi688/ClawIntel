## 注意事项：避坑指南

### 坑1：过度配置

**症状：**
- 一开始就让 AI 创建 10+ 个配置文件
- 每个文件都配置得很详细
- 系统越来越复杂，越来越难维护

**解决方案：**
- 从基础开始，让 AI 创建最少必需的文件
- 根据实际使用需求，再让 AI 添加
- 保持系统简单可维护

**你只需要：** 跟 AI 说"先从最简单的开始"

### 坑2：追求完美

**症状：**
- 花大量时间优化提示词
- 总觉得不够好，一直改
- 迟迟开始使用

**解决方案：**
- 先做个"能用的版本"
- 在使用中持续迭代
- 快速试错，从反馈中优化

**核心原则：** 用起来比完美更重要。

### 坑3：忽视成本

**症状：**
- 盲目使用付费 API
- 追求最新最贵的模型
- 月费越来越高

**解决方案：**
- **优先使用免费模型**（OpenCode + GLM-4.7）
- Claude Code 账户自带免费额度
- 等有收益后再考虑付费升级

### 坑4：忘记备份

**症状：**
- 配置文件只存在本地
- 电脑坏了就全没了
- 换电脑很麻烦

**解决方案：**
- 用 GitHub 托管配置
- 用 Obsidian Git 插件自动备份
- 跨电脑同步（工作时/家里时）

## 进阶使用：Skill 系统

当你已经搭建好基础的数字分身系统，并且用了一段时间后，你可能会发现：

**有些事情是重复的，有些工作流程是固定的。**

比如：
- 每次都要跟 AI 解释"我的公众号配图要什么风格"
- 每次都要告诉 AI "我的语速是每分钟 200 字"
- 每次都要说明"我的视频时长控制在 60 秒内"

**这时候，就可以考虑引入 Skill 系统。**

### 什么是 Skill？

**说人话：Skill 就是把你的"工作经验"打包成文件。**

数字分身记住的是"你是谁"，而 Skill 记住的是"你怎么干活"。

**打个比方：**
- 数字分身 = 你的大脑（认识你、了解你）
- Skill = 你的专业技能（会写文章、会做图、会剪视频）

### 什么时候需要 Skill？

**判断标准：**

✅ **需要引入 Skill 的情况：**
- 发现自己在重复解释某些东西
- 有固定的创作风格和工作流程
- 希望一键完成某个复杂任务
- 想把经验分享给别人

❌ **不需要 Skill 的情况：**
- 刚开始搭建数字分身
- 需求还在不断变化
- 还没有固定的工作流程

**核心原则：** 先把基础系统用起来，Skill 可以慢慢加。
