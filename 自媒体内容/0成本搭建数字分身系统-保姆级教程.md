## 注意事项：避坑指南

### 坑1：过度配置

**症状：**
- 一开始就让 AI 创建 10+ 个配置文件
- 每个文件都配置得很详细
- 系统越来越复杂，越来越难维护

**解决方案：**
- 从基础开始，让 AI 创建最少必需的文件
- 根据实际使用需求，再让 AI 添加
- 保持系统简单可维护

**你只需要：** 跟 AI 说"先从最简单的开始"

### 坑2：追求完美

**症状：**
- 花大量时间优化提示词
- 总觉得不够好，一直改
- 迟迟开始使用

**解决方案：**
- 先做个"能用的版本"
- 在使用中持续迭代
- 快速试错，从反馈中优化
- 让你的

### 坑3：忽视成本

**症状：**
- 盲目使用付费 API
- 追求最新最贵的模型
- 月费越来越高

**解决方案：**
- **优先使用免费模型**（OpenCode + Qwen/Google 免费额度）
- Claude Code 账户自带免费额度
- 等有收益后再考虑付费升级

### 坑4：忘记备份

**症状：**
- 配置文件只存在本地
- 电脑坏了就全没了
- 换电脑很麻烦

**解决方案：**
- 用 GitHub 托管配置
- 用 Obsidian Git 插件自动备份
- 跨电脑同步（工作时/家里时）

## 进阶使用：Skill 系统

当你已经搭建好基础的数字分身系统，并且用了一段时间后，你可能会发现：

**有些事情是重复的，有些工作流程是固定的。**

比如：
- 每次都要跟 AI 解释"我的公众号配图要什么风格"
- 每次都要告诉 AI "我的语速是每分钟 200 字"
- 每次都要说明"我的视频时长控制在 60 秒内"

**这时候，就可以考虑引入 Skill 系统。**

### 什么是 Skill？

**说人话：Skill 就是把你的"工作经验"打包成文件。**

数字分身记住的是"你是谁"，而 Skill 记住的是"你怎么干活"。

**打个比方：**
- 数字分身 = 你的大脑（认识你、了解你）
- Skill = 你的专业技能（会写文章、会做图、会剪视频）

### 什么时候需要 Skill？

**判断标准：**

✅ **需要引入 Skill 的情况：**
- 发现自己在重复解释某些东西
- 有固定的创作风格和工作流程
- 希望一键完成某个复杂任务
- 想把经验分享给别人

❌ **不需要 Skill 的情况：**
- 刚开始搭建数字分身
- 需求还在不断变化
- 还没有固定的工作流程

**核心原则：** 先把基础系统用起来，Skill 可以慢慢加。

### Skill 的威力

**举个具体的例子：**

**场景1：自媒体配图 Skill**

**没有 Skill：**
```
你：帮我生成一张公众号配图
AI：什么风格？什么尺寸？什么色系？
你：简约插画风格，900*500，蓝色系...
AI：好的，生成中...
```

**有 Skill：**
```
你：帮我生成一张公众号配图
AI：[按照你预设的风格、尺寸、色系，直接生成]
```

Skill 里保存的内容：
- 你的品牌色系（蓝色渐变）
- 喜欢的图片风格（扁平插画）
- 标准尺寸（公众号首图 900*500）
- 文字排版规则（标题居中、16px）

---

**场景2：口播文案生成 Skill**

**没有 Skill：**
```
你：帮我写一个口播文案，主题是 AI 工具
AI：多少字？什么开头？什么结尾？
你：60秒左右，开头要吸引人，结尾要引导关注...
AI：好的，正在写...
```

**有 Skill：**
```
你：帮我写一个口播文案，主题是 AI 工具
AI：[按照你的语速、风格、套路，直接生成]
```

Skill 里保存的内容：
- 你的语速（每分钟 200 字）
- 开场白套路（"今天给大家分享一个超好用的..."）
- 常用的口头禅（"说句大实话"、"重点是"）
- 视频时长限制（60 秒内）

---

**场景3：数字分身激活 Skill**

**没有 Skill：**
```
你：读取我的背景文件，汇报一下
AI：正在读取 personal_background.md...
AI：正在读取 task_backlog.json...
AI：正在读取 daily_reports.json...
```

**有 Skill：**
```
你：激活数字分身
AI：[自动读取所有背景文件，直接向你汇报]
```

Skill 里保存的内容：
- 读取哪些文件（personal_background.md、conversation_log.md、task_backlog.json）
- 汇报什么内容（昨天完成、今天待办、优先级建议）
- 汇报格式（简洁明了，分点列出）

### 如何创建 Skill？

**简单说：就是在 `~/.claude/skills/` 目录创建一个文件夹，写一个 SKILL.md 文件。**

**具体步骤：**

1. 打开终端，输入：
```bash
mkdir -p ~/.claude/skills/my-skill
```

2. 创建 SKILL.md 文件：
```bash
nano ~/.claude/skills/my-skill/SKILL.md
```

3. 写入你的技能定义（告诉 AI 这个技能是做什么的）

4. 保存后，重启 OpenCode

5. 以后直接调用这个 Skill

**💡 但其实更简单的方式是：**

跟 AI 说："我想创建一个 Skill，功能是 xxx，帮我生成 SKILL.md 文件"

AI 会自动帮你创建好文件。

### 💡 小贴士

**建议把"数字分身激活"也做成 Skill。**

这样你只需要说一句话"激活数字分身"，AI 就会自动读取所有背景文件并向你汇报。

**但记住：** 先把基础系统搭起来，Skill 可以慢慢加。别一开始就折腾太多。

**A：非常安全。**

**为什么安全？**

1. **数据在本地** - 所有配置文件都在你电脑里，没人能拿走
2. **GitHub 备份** - 用 GitHub 自动备份，比任何网盘都稳定

**A：超简单，两步搞定。**

**完成。**

这就是用 GitHub 的好处。所有配置都在云端，换电脑、换手机，随时同步。

### Q6：Skill 真的需要吗？

**A：强烈推荐，但不是强制。**

**没有 Skill 的问题：**
- 每次都要手动输入一堆指令（麻烦）
- 换电脑要重新配置（容易忘）

**有 Skill 的好处：**
- 一句话激活，超简单
- 多端同步，换电脑也能用
- 越用越顺手，形成习惯

**我的建议：** 先把基础系统搭起来，熟悉了再装 Skill。别一开始就折腾太多。

### Q7：Obsidian 必须吗？

**A：建议用，但不是强制。**

**用 Obsidian 的好处：**
- 免费、强大、本地存储
- 双向链接、反向链接、全文搜索
- 插件生态丰富（堪比 Chrome）
- 跨平台同步（Mac / Windows / 手机）

**不用也行：**
- 用 VS Code + 插件（适合程序员）
- 用 Notion（但数据在别人服务器上）
- 用其他笔记工具（都可以，只要能存 Markdown）

**核心是：** 你需要一个地方存储知识。Obsidian 是目前最好的选择，但不是唯一选择。
